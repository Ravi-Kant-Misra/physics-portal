Graphing with Logger Pro

Enter data, label axes, set error bars, add best-fit lines, and create calculated columns — the complete IB Physics workflow

💻

What is Logger Pro?

Logger Pro is a data analysis application used in IB Physics labs. It lets you enter experimental data into a spreadsheet-style table, automatically plots points on a graph, and provides tools for adding error bars, drawing best-fit lines, and creating new calculated columns from your data.

When you open Logger Pro you see two panels: a Data Set table on the left (where you enter X and Y values) and a graph on the right that auto-updates as you type. The default columns are labelled X and Y — you will rename these to match your experiment.

📝

Step-by-Step: Entering and Labelling Data

1

Open Logger Pro — default view

You see a blank data table (columns X and Y) and an empty graph. The graph axes are labelled X and Y until you rename them. No device needs to be connected for manual data entry.

2

Rename the X column — double-click the header

Double-click on the "X" column header to open Manual Column Options. Fill in:

Name: Time  |  Short Name: t  |  Units: s

The graph x-axis will automatically update to show "Time (s)".

3

Rename the Y column — double-click the header

Double-click the "Y" header and rename to match your DV. Example:

Name: Position  |  Short Name: p  |  Units: m

The y-axis label updates to "Position (m)".

4

Enter your data — type into the cells

Click a cell and type values. Press Enter to move to the next row. As you enter data, points appear on the graph automatically.

Adding Error Bars

How to Set Error Bars in Logger Pro

Go to Data → Column Options (or double-click a column header and go to the Options tab). Look for the Error Bar Calculations section. You can set errors as:

Percentage
Fixed Value
Fixed Value (from another column)

For IB Physics you typically use Fixed Value and type in your error bar value from your data table. X-axis error bars come from your IV uncertainty; Y-axis error bars come from the Error Bar column in your data table.

± 0.002

X-axis vs Y-axis error bars in Logger Pro

Set the X column error bar = your IV measurement uncertainty. Set the Y column error bar = the Error Bar column value from your data table (the larger of measurement and statistical uncertainty). If all rows have the same error bar, use Fixed Value. If they differ, you may need a separate error bar column.

Best-Fit Lines — Min, Max, and Best

In IB Physics you draw three lines of best fit through your error bars: a minimum slope line, a maximum slope line, and a best-fit line. These are used to calculate the uncertainty in the gradient (slope) and y-intercept of your graph.

In Logger Pro: go to Analyze → Linear Fit to add the best-fit line. The software shows the equation (slope m, y-intercept b, and uncertainty in each). You then manually draw the min and max lines through the extremes of the error bars.

Best-Fit Line

Passes through the middle of the data — the line that minimises the distance to all points. Logger Pro's Linear Fit gives this automatically with slope, intercept, and R² value.

Minimum Slope Line

Drawn with the smallest possible gradient that still passes through all error bars. Connects the top of the first point's error bar to the bottom of the last point's error bar.

Maximum Slope Line

Drawn with the largest possible gradient that still passes through all error bars. Connects the bottom of the first point's error bar to the top of the last point's error bar.

Calculating gradient uncertainty from min/max lines

Δgradient = (max slope − min slope) / 2. This gives you the uncertainty in the gradient of your best-fit line — an essential part of your IB lab analysis.

🧮

Calculated Columns

Creating a New Calculated Column

Go to Data → New Calculated Column. This opens a dialog where you can define a new column whose values are calculated automatically from existing columns using a formula.

Use the Variables (Columns) and Functions buttons to insert column names and mathematical functions into your expression.

"Force 1" = "mass" * 9.81
"v²" = "velocity"^2
"1/T²" = 1 / "Period"^2

This is powerful for linearising graphs — for example, if you suspect y ∝ x², create a column for x² and plot y vs x². If the relationship is correct, you get a straight line.

Why linearise? A straight-line graph is much easier to analyse than a curve. By plotting the right combination of variables (e.g. T² vs L for a pendulum instead of T vs L), you get a straight line whose gradient directly gives you a physical constant. Logger Pro's Calculated Column feature makes this quick and accurate.
